# Diagnostics Polish Execution Workspace

## Purpose

This workspace holds process material for the diagnostics-polish effort after
the design spec was promoted into `docs/diagnostics_polish_spec.md`.

Its job is to track:

- stage decomposition
- acceptance markers
- progress checkpoints
- review surfaces for main-agent and sub-agent work

## Explicit Boundary

This workspace is not a human-style time schedule.

Do not optimize it for:

- calendar dates
- weekly planning language
- staffing charts
- vague priority prose

Instead, optimize it for:

- clear stage boundaries
- objective acceptance criteria
- traceable progress state
- easy QA/review by the primary agent

## Required Output Shape

Any serious artifact in this workspace should make clear:

1. what each stage is for
2. what must be true before that stage counts as complete
3. what evidence should exist for review
4. what regressions or false progress signals to watch for

## Current Anchor

The live design authority is now:

- `docs/diagnostics_polish_spec.md`

This workspace should operationalize that specification without turning back
into a design debate.
