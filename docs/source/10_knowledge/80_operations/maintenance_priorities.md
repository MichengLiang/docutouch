# Maintenance Priorities

## 作用域

本页记录 DocuTouch 当前已接纳的维护优先级排序。

它回答：

- 维护时应先保什么
- 什么问题属于更高优先级的基础层问题
- 哪些变化必须同步到文档面

## 主路径优先

维护优先级首先围绕高频主路径展开：

1. 定位工作区
2. 查看目录
3. 读取上下文
4. 应用结构化 patch
5. 获取可诊断反馈

若某项工作不能明显改善这条主路径，
则不应仅凭“新增能力”而获得优先级。

## 基础层问题优先于扩功能

下列问题优先按基础层问题处理：

- 路径正确性
- 事务边界
- 回滚一致性
- 输出结构稳定性
- 跨平台差异
- warning / error 的可解释性

这类问题通常比扩新命令更值得优先修复。

## 文档同步义务

当以下变化发生时，不能只改代码：

- 产品定位变化
- 主接口 contract 变化
- warning / error 策略变化
- 与上游基线的关系变化
- 工具边界变化
- 近期优先级变化
- 对主形态的判断变化

## 长期维护纪律

- 不把临时聊天结论当成长期事实
- 会影响未来维护的判断，必须沉淀为稳定文档对象
- 不让零散过程材料替代当前可依赖知识
