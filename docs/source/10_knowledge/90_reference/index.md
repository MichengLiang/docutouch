# 90 Reference

## 作用域

本目录承载 accepted knowledge tree 中的派生支撑面。

它回答：哪些 reference-like surface 需要被保留，但不应反向污染主体知识家族。

## 典型对象

- domain-level glossary
- concept inventory
- notation guide
- appendix-like support pages
- crosswalks
- reference tables

## 不承担的职责

- 不做主体对象的唯一 authority source
- 不承载未收敛 glossary
- 不承载 records/inventory

## Dependency Position

### Upstream Dependencies

可从各主体家族派生整理。

### Downstream Dependents

通常不作为主体家族的 authority 上游。

### Lateral Cross-References

几乎所有主体家族都可指向 `reference/` 作为辅助 lookup surface，但不应把 reference 误当成主体定义宿主。

### Backlink Rule

所有派生性的 reference 页面，都应显式指回其 canonical subject-family source，而不能只保留整理后的派生表面。
