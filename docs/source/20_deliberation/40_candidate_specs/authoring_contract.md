# 40 Candidate Specs 作者契约

## 契约范围

本页裁定哪些对象已长成 candidate specification family。

## Allowed Objects

- candidate spec with structured sections
- candidate contract surface
- candidate architecture / operations / requirements surface

## Disallowed Objects

- high-level proposal without specification form
- accepted spec
- generic issue
- worklist item

## Dependency Discipline

- candidate spec 页必须说明其 target accepted family
- candidate spec 是 promotion 的默认直接前置态；若对象尚未长成可 hand-off 的结构，不应提前进入本 family
- candidate spec 不得长期被下游当成 accepted knowledge 使用而仍停留本目录
- 若对象被 accepted，原页应改写为指针或处置说明，不继续承担影子 authority
- promotion 不等于简单挪目录；若进入 accepted knowledge，应同步产生或刷新相应 records bundle
